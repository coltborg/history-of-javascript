The point of it all

The advent of Node and the maturation of Javascript as a language and its community is:
  • Making front-end development more interesting, fun, and efficient
  • Providing front-end-specific package/dependency management where there once was none
  • Encouraging layered architecture

This presentation is NOT

  • Suggesting we use JS to do Java or Ruby's jobs
  • About JS running on an application server at all
  • Me trying to convert you to JS evangelists

This presentation IS

  • Hoping to ignite some curiousity about JS
  • Five parts
    1. Recommending a discreet front end
    2. Intro to Node
    3. Intro to NPM
    4. Intro to Bower
    5. Intro to GruntJS

What is NodeJS?

Node.jsÂ® is a JavaScript runtime built on Chrome's V8 JavaScript engine.
  • Open source and cross-platform
  • Event driven (EventEmitter API)
  • Single threaded
  • Non-blocking I/O
  • Uses libuv under the hood which manages queueing, the event loop, workers, thread pooling, mem alloc, etc...
  • Includes NPM
  • Checkout the NodeJS website

What is NPM?

Installs, publishes and manages node programs
  • Node Package Manager
  • Uses SemVer (~, ^, =, <, >, <=, >=)
  • NPM hosts the world's largest repository of modules, in any language
    • Filesystem traversal & manipulation
    • Executing shell scripts & commands
    • Web application frameworks
    • Webservers, process managers, load balancers, etc...

Dependency Injection

  • Node knows to look in node_modules directory
  • You can also specify a path to a file

Dependency Injection Cont.

  1. Is it a "core" module (http, fs, etc...)?
  2. Is it in local node_modules dir?
  3. Traverse upwards to global packages dir

  • Built on Node
  • Grunt competes with Gulp
  • Automates your tasks so you don't have to
    • Compilation

    • Compression
